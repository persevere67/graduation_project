# 半同态加密原型设计说明

## 1. 设计目标

为了进一步增强联邦学习场景下模型参数传输过程的隐私保护能力，本文设计了一个面向参数聚合过程的半同态加密原型。该原型并不追求完整的工程部署，而是用于验证以下核心思想：

1. 客户端可以在本地对模型更新进行加密；
2. 服务器在不解密单个客户端参数的前提下，可直接对密文执行加法聚合；
3. 聚合结果在解密后应与明文直接求和结果一致。

这一原型可作为“安全聚合机制”的概念验证，为后续将更严格的密码学保护机制引入联邦推荐系统提供参考。

## 2. 原型方法

本文原型采用 `Paillier` 加法同态加密的教学型实现。Paillier 方案的核心特点在于：

- 明文加法对应密文乘法；
- 多个客户端的参数更新可以分别加密；
- 服务器对密文逐项相乘后，相当于完成了明文求和；
- 解密聚合密文后即可得到与明文直接聚合一致的结果。

在原型演示中，客户端上传的并非完整高维模型参数，而是使用三个低维向量模拟局部模型更新，从而突出“密文聚合”的核心思想。

## 3. 原型脚本

原型脚本位置：

- [he_prototype.py](/F:/ graduation_project/experiments/he_prototype.py:1)

运行方式：

```bash
python experiments/he_prototype.py
```

脚本将完成以下流程：

1. 生成一组简化的 Paillier 公私钥；
2. 构造 3 个客户端的模拟参数更新向量；
3. 对每个客户端更新向量逐元素编码并加密；
4. 由服务器在密文域中完成逐元素加法聚合；
5. 对聚合结果解密并与明文直接求和结果进行比对。

## 4. 原型意义

该原型的意义主要体现在以下两个方面：

- 从概念上证明了“参数密文聚合”的可行性；
- 为论文后续讨论“安全聚合与隐私增强的进一步扩展方向”提供了技术支撑。

需要强调的是，当前实现属于教学演示型原型，主要用于毕设论文中的方案说明，不应视为生产环境可直接部署的安全系统。实际工程中还需要进一步考虑大整数运算效率、密钥分发管理、客户端掉线、鲁棒聚合以及多方安全计算等问题。
